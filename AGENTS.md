# AGENTS.md — macos-widgets-stats-from-website

Conventions for any AI agent (Claude Code, OpenClaw, Codex, etc.) working in
this repo. Project-level rules take precedence over global CLAUDE.md.

## Version: single source of truth

The marketing version + Sparkle build number live in **one place**:

```yaml
# project.yml
settings:
  base:
    MARKETING_VERSION: "0.12.10"        # CFBundleShortVersionString
    CURRENT_PROJECT_VERSION: "1208"     # CFBundleVersion
```

Everything else inherits:

- `Info.plist` files carry `$(MARKETING_VERSION)` / `$(CURRENT_PROJECT_VERSION)`
  placeholders — xcodebuild substitutes the literal values at build time.
  These plists are **regenerated by xcodegen** from the `info.properties`
  block in `project.yml`. **Never hand-edit Info.plist version fields.**
- Swift code reads `Bundle.main.infoDictionary?["CFBundleShortVersionString"]`
  at runtime (see `MCPServer.marketingVersion` and `Apps/CLI/main.swift`).
- `scripts/prepare_release_metadata.py` reads MARKETING_VERSION from
  project.yml, computes the release build number, and patches Info.plists
  for the CI artifact only (the working tree stays placeholder-form).
- `scripts/validate_release_metadata.py` enforces that Info.plists hold the
  literal `$(MARKETING_VERSION)` / `$(CURRENT_PROJECT_VERSION)` placeholders
  and that no per-target literal version values have crept back into
  project.yml.

## Per-commit version bump

Every commit that touches code or asset files is expected to bump
MARKETING_VERSION (patch by default). Use the helper:

```bash
./scripts/bump-version.sh           # patch bump + xcodegen
./scripts/bump-version.sh minor
./scripts/bump-version.sh set 0.13.0
```

The bump helper increments `CURRENT_PROJECT_VERSION` by one as well, so
Sparkle's monotonic-build-number invariant holds.

## Pre-commit hook

The repo ships a tracked `.githooks/pre-commit` that fails the commit if:

- any code/asset file is staged, AND
- `MARKETING_VERSION` in `project.yml` matches the previous commit's value.

Install once per clone:

```bash
git config core.hooksPath .githooks
```

Documentation-only commits (README, docs/, .github/, AGENTS.md, scripts/
release helpers) are exempt from the bump check. To bypass for a genuine
non-shipping change set `SKIP_VERSION_CHECK=1` in the environment.

## Build flow

`./scripts/build.sh` wraps `xcodegen` + `xcodebuild` for local Debug builds.
Important flags:

- `DEV_RESIGN=1` (or `--dev-resign`) — applies the chronod-resign workaround
  so the widget extension registers on personal/dev machines. Required when
  testing widget configuration locally; do **not** modify the DEV_RESIGN
  block — it carries an extensive comment explaining the chronod policy.
- `ALLOW_PROVISIONING_UPDATES=1` — passes `-allowProvisioningUpdates` to
  xcodebuild for first-run signing setup.

Standard manual build:

```bash
xcodegen
DEV_RESIGN=1 ALLOW_PROVISIONING_UPDATES=1 ./scripts/build.sh
```

## Other conventions

- Per global CLAUDE.md: UI verification uses **native** `screencapture -l <windowID>`,
  not Playwright.
- The widget extension's first-launch app-group migration logic in commit
  `5e82282`, the personal-Chrome regression fix in `d6ff449`, and the
  prior-instance kill in `0ed19df` are load-bearing — leave them alone
  unless the change explicitly targets them.
- Release flow: `.github/workflows/release.yml` validates metadata, runs
  `prepare_release_metadata.py --apply-plists`, archives + notarizes, signs
  with Sparkle ED25519, and updates `gh-pages/appcast.xml`. Don't bypass.
